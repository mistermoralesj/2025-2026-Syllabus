Course Overview

AP Computer Science A is a college-level course that introduces students to the fundamentals of computer science through programming in Java. The course emphasizes problem-solving, algorithm development, and object-oriented programming. Students will learn to design, implement, test, and analyze solutions to real-world problems using industry-standard programming practices.

Throughout the year, students will develop proficiency in Java syntax and structure, work with control structures, arrays, ArrayLists, and object-oriented class design. Students will also focus on writing readable, reusable, and efficient code that demonstrates a strong understanding of computational thinking.

In addition to preparing students for the AP Computer Science A Exam, this course also prepares students to pursue the Certiport ITS Java Certification, offering them an additional credential that demonstrates their programming competence and industry readiness.

Prerequisites:
  • A prior course or experience in programming is strongly recommended before enrolling.
    • Ideal preparation includes AP Computer Science Principles, Digital Information Technology (taken at NeoCity), Applied Engineering Technology 2, or another high school-level coding course.
    • Please note: Students who completed Digital Information Technology in middle school or at another high school do not meet the prerequisite for this course.
  • This course is not recommended for complete beginners, as it moves quickly and assumes a foundational understanding of programming concepts.

Unit Overview

Unit Title Description
1 Getting Started with Java Students explore the Java language through variables, data types, expressions, and console input/output. Emphasis is placed on translating prior programming experience into Java's stricter syntax and object-based approach.
2 Methods and Objects Students explore object-oriented programming concepts such as method design, object instantiation, instance methods, and String manipulation.
3 Conditionals and Boolean Logic Students develop control flow using Boolean expressions, if-statements, and logical comparisons, building on their foundation in Boolean Algebra.
4 Iteration and Performance Students implement repetition using while, for, and nested loops, then evaluate algorithm efficiency and runtime performance.
5 Class Design and Abstraction Students design Java classes from scratch, write constructors, organize methods, manage object references, and use the this keyword to reinforce encapsulation and abstraction.
6 Inheritance and Polymorphism Students explore how inheritance supports code reuse and structure, and how polymorphism enables flexible design using superclass references and overridden methods.
7 Arrays and ArrayLists Students learn to use one-dimensional arrays and ArrayLists, focusing on traversal, search, and manipulation through common data algorithms.
8 2D Arrays, Sorting, and Recursion Students extend their skills to 2D arrays, apply recursion, and implement classic searching and sorting algorithms.
9 Java Certification Prep Students prepare for the Java certification exam through structured review, syntax practice, and debugging tasks.
10 AP Exam Prep Students prepare for the AP CSA exam through timed practice, free-response strategies, and multiple-choice review.
11 Graphics Students learn to build interactive programs using graphical libraries such as java.awt and javax.swing.

  • I – Involved: Participate meaningfully, be present, and contribute to class activities.

Academic Integrity Policy

NeoCity Academy maintains high standards of honesty, curiosity, and independent thinking. Students must demonstrate academic integrity in all work; every submission—program, reflection, report, or project—must represent the student's own thinking and effort.

Key Expectations

  • Do your own work: All submissions must be original. When in doubt, ask Mr. Morales before turning in an assignment.
  • Do not share or post answers: Never post solutions to assignments, projects, or assessments online, even after the due date.
  • Build portfolios responsibly: Use Canvas portfolios to keep work private; avoid public platforms such as GitHub.
  • Collaboration limits: Discuss concepts with classmates, but submit individual work.
  • Use resources ethically: Cite sources and avoid copying full solutions. AI tools may be used only when explicitly permitted.
  • Respect assessments: Quizzes and exams must be completed independently. Do not discuss specific questions or answers with classmates.

Examples of Acceptable Behavior

  • Asking a classmate to clarify a concept (e.g., how a loop works)
  • Discussing the meaning of an error message
  • Using outside resources to clarify a topic (with citation)
  • Debugging code with peer assistance without copying
  • Requesting clarification or guidance from Mr. Morales

Examples of Unacceptable Behavior

  • Submitting code, text, or data written by someone else or by AI (see AI policy)
  • Posting code or solutions in a public space: Sharing work where others can access it is prohibited. Use Canvas portfolios and consult Mr. Morales before sharing work publicly.
  • Viewing another student's code before submitting your own: Only compare code after both submissions are complete and with permission.
  • Splitting an individual assignment and submitting combined results: Not allowed unless designated as a group project.
  • Reusing work from another class or school without permission: Submitting previous work without approval is self-plagiarism.
  • Sharing assessment answers or questions: Even minor hints give an unfair advantage. Discuss assessments only with Mr. Morales.

Bottom Line: All work must be original. Violations may result in reduced or no credit, an academic-integrity report, or further disciplinary action. If uncertain, ask Mr. Morales before submitting.

Artificial Intelligence (AI) Use Policy

"Use AI as a step, not as a crutch."
— Mr. Morales

The school district blocks access to AI tools such as ChatGPT, GitHub Copilot, and Bing AI on campus. Students may use these tools outside of school hours.

DO NOT use VPNs or other workarounds to access AI tools on campus. At no point has Mr. Morales authorized the use of a VPN for this purpose. Mr. Morales will not advocate for or excuse violations of school-district technology policies.

If unsure whether AI use is appropriate, ask Mr. Morales first; he is available to help students make responsible, ethical choices.

Permitted AI Uses

  • Clarifying difficult concepts
  • Breaking down and exploring new topics
  • Studying for quizzes or exams
  • Reviewing JavaScript syntax
  • Debugging code (students must also show code versions before using AI to demonstrate original work)

Prohibited AI Uses

  • Generating full solutions for graded work
  • Writing code that is then submitted as original work
  • Answering reflection or documentation prompts for assignments

Improper AI use may result in a zero on the assignment, contacting parent/guardian, or more severe disciplinary action. Students are encouraged to use AI to learn more effectively, not to bypass the learning process.
