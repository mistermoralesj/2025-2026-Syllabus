Course Overview

This year-long course introduces students to foundational computer science and IT skills through hands-on coding, system management, and web development. The course is organized by quarter:

  • Quarter 1: HTML and CSS for web development, including structure, styling, and certification prep
  • Quarter 2: Device hardware, system configuration, and networking, leading to industry certification
  • Quarters 3 & 4: Python programming, covering logic, data structures, and software development techniques

Students will prepare for and attempt the following industry certifications:

  • ITS HTML and CSS – required in Quarter 1
  • ITS Device Configuration – required in Quarter 2

This course is the first in the Artificial Intelligence pathway and provides essential background for AP Computer Science A.

Prerequisites:
Students must have successfully completed Algebra I. No prior programming experience is required.

Unit Overview

Unit Title Description
0 Typing Skills Daily typing practice to improve fluency and accuracy.
1 HTML & Page Structure Use HTML to create structured web content, including text, links, lists, and tables.
2 Styling with CSS Apply styles using CSS to control layout, fonts, colors, and responsive design.
3 Cert Prep: ITS HTML & CSS Review core HTML/CSS concepts and take the industry certification exam.
4 Hardware & Networking Explore internal hardware components, peripheral devices, and basic networking principles.
5 System Configuration Learn to set up operating systems, manage users, and configure basic system settings.
6 Cert Prep: ITS Device Configuration Prepare for and take the ITS Device Configuration certification exam.
7 Python Fundamentals Write Python programs using input/output, variables, data types, and basic expressions.
8 Control Structures Use `if`, `else`, loops, and Boolean logic to control program flow and automate decisions.
9 Functions & Decomposition Define and call functions with parameters and return values to structure reusable code.
10 Boolean Algebra Analyze symbolic logic using truth tables, logic gates, and Boolean expressions.
11 Data Structures Work with lists, dictionaries, strings, and nested data to store and manipulate information.
12 Python Project Design and develop a complete Python program that demonstrates full-cycle problem-solving.

  • R – Responsible: Own choices, actions, and impact—in class, in conversations, and in learning.
  • I – Involved: Participate meaningfully, be present, and contribute to class activities.

Academic Integrity Policy

NeoCity Academy maintains high standards of honesty, curiosity, and independent thinking. Students must demonstrate academic integrity in all work; every submission—program, reflection, report, or project—must represent the student's own thinking and effort.

Key Expectations

  • Do your own work: All submissions must be original. When in doubt, ask Mr. Morales before turning in an assignment.
  • Do not share or post answers: Never post solutions to assignments, projects, or assessments online, even after the due date.
  • Build portfolios responsibly: Use Canvas portfolios to keep work private; avoid public platforms such as GitHub.
  • Collaboration limits: Discuss concepts with classmates, but submit individual work.
  • Use resources ethically: Cite sources and avoid copying full solutions. AI tools may be used only when explicitly permitted.
  • Respect assessments: Quizzes and exams must be completed independently. Do not discuss specific questions or answers with classmates.

Examples of Acceptable Behavior

  • Asking a classmate to clarify a concept (e.g., how a loop works)
  • Discussing the meaning of an error message
  • Using outside resources to clarify a topic (with citation)
  • Debugging code with peer assistance without copying
  • Requesting clarification or guidance from Mr. Morales

Examples of Unacceptable Behavior

  • Submitting code, text, or data written by someone else or by AI (see AI policy)
  • Posting code or solutions in a public space: Sharing work where others can access it is prohibited. Use Canvas portfolios and consult Mr. Morales before sharing work publicly.
  • Viewing another student's code before submitting your own: Only compare code after both submissions are complete and with permission.
  • Splitting an individual assignment and submitting combined results: Not allowed unless designated as a group project.
  • Reusing work from another class or school without permission: Submitting previous work without approval is self-plagiarism.
  • Sharing assessment answers or questions: Even minor hints give an unfair advantage. Discuss assessments only with Mr. Morales.

Bottom Line: All work must be original. Violations may result in reduced or no credit, an academic-integrity report, or further disciplinary action. If uncertain, ask Mr. Morales before submitting.

Artificial Intelligence (AI) Use Policy

"Use AI as a step, not as a crutch."
— Mr. Morales

The school district blocks access to AI tools such as ChatGPT, GitHub Copilot, and Bing AI on campus. Students may use these tools outside of school hours.

DO NOT use VPNs or other workarounds to access AI tools on campus. At no point has Mr. Morales authorized the use of a VPN for this purpose. Mr. Morales will not advocate for or excuse violations of school-district technology policies.

If unsure whether AI use is appropriate, ask Mr. Morales first; he is available to help students make responsible, ethical choices.

Permitted AI Uses

  • Clarifying difficult concepts
  • Breaking down and exploring new topics
  • Studying for quizzes or exams
  • Reviewing JavaScript syntax
  • Debugging code (students must also show code versions before using AI to demonstrate original work)

Prohibited AI Uses

  • Generating full solutions for graded work
  • Writing code that is then submitted as original work
  • Answering reflection or documentation prompts for assignments

Improper AI use may result in a zero on the assignment, contacting parent/guardian, or more severe disciplinary action. Students are encouraged to use AI to learn more effectively, not to bypass the learning process.
