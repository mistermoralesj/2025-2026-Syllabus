Course Overview

This year-long course introduces students to foundational computer science concepts through programming, problem-solving, and systems-level thinking. The first semester focuses on developing typing fluency, computational thinking, and programming skills using Python. Students explore variables, input/output, control structures, Boolean logic, loops, functions, data abstraction, and pattern recognition. At the end of the semester students prepare for and take the ITS Computational Thinking certification exam.

In the second semester, students transition to the physical and structural components of computing. Topics include computer hardware, system software, networking, IP addressing, and system configuration. Students then move into relational databases and data models, learning how to design schemas, use entity-relationship diagrams, and query data using SQL. The course concludes with preparation for the ITS Device Configuration certification exam.

This course serves as the foundation of the four-year Artificial Intelligence Pathway, emphasizing hands-on experience, conceptual understanding, and interdisciplinary thinking. It prepares students for deeper exploration of intelligent systems and is a required prerequisite for the next course in the pathway as well as a recommended prerequisite for AP Computer Science A.

Prerequisites:
No prior coding experience is required. Students should have successfully completed Algebra I and be comfortable working with linear equations and basic mathematical reasoning.

Unit Overview

Unit Title Description
0 Typing Skills Daily practice to improve typing fluency and accuracy.
1 Intro to Python & Computing Explore how code interacts with computer systems using input, output, and basic data types.
2 Logic & Control Flow Use Boolean logic, conditional statements, and loops to automate decision-making and control program flow.
3 Functions Define and call functions, use parameters and return values, and structure code for reusability.
4 Data Abstraction Work with lists, tuples, strings, and dictionaries to organize and manipulate structured data.
5 Boolean Algebra Connect logic and truth tables to symbolic expressions and digital reasoning.
6 Intro to Data Science Practice basic data analysis using Python, including cleaning and summarizing datasets with Pandas and NumPy.
7 Cert Prep: ITS Computational Thinking Review and prepare for the ITS Computational Thinking certification exam.
8 Hardware & Networking Fundamentals Understand computer components, peripheral devices, and basic networking concepts.
9 Addressing & Connectivity Learn IP addressing, subnetting, and the basics of wired and wireless network configuration.
10 System Configuration & Management Explore operating systems, user accounts, software management, and system updates.
11 Cert Prep: ITS Device Configuration Final preparation and first attempt at the ITS Device Configuration certification exam.
12 Relational Databases & Data Models Design relational schemas, use ER diagrams, and understand primary/foreign key relationships.
13 Querying & Managing Data with SQL Write SQL queries, use JOINs, and manage relational data.
14 Cert Prep: ITS Databases Review data modeling and SQL concepts and prepare for the ITS Database certification exam.

Academic Integrity Policy

NeoCity Academy maintains high standards of honesty, curiosity, and independent thinking. Students must demonstrate academic integrity in all work; every submissionâ€”program, reflection, report, or projectâ€”must represent the student's own thinking and effort.

Key Expectations

  • Do your own work: All submissions must be original. When in doubt, ask Mr. Morales before turning in an assignment.
  • Do not share or post answers: Never post solutions to assignments, projects, or assessments online, even after the due date.
  • Build portfolios responsibly: Use Canvas portfolios to keep work private; avoid public platforms such as GitHub.
  • Collaboration limits: Discuss concepts with classmates, but submit individual work.
  • Use resources ethically: Cite sources and avoid copying full solutions. AI tools may be used only when explicitly permitted.
  • Respect assessments: Quizzes and exams must be completed independently. Do not discuss specific questions or answers with classmates.

Examples of Acceptable Behavior

  • Asking a classmate to clarify a concept (e.g., how a loop works)
  • Discussing the meaning of an error message
  • Using outside resources to clarify a topic (with citation)
  • Debugging code with peer assistance without copying
  • Requesting clarification or guidance from Mr. Morales

Examples of Unacceptable Behavior

  • Submitting code, text, or data written by someone else or by AI (see AI policy)
  • Posting code or solutions in a public space: Sharing work where others can access it is prohibited. Use Canvas portfolios and consult Mr. Morales before sharing work publicly.
  • Viewing another student's code before submitting your own: Only compare code after both submissions are complete and with permission.
  • Splitting an individual assignment and submitting combined results: Not allowed unless designated as a group project.
  • Reusing work from another class or school without permission: Submitting previous work without approval is self-plagiarism.
  • Sharing assessment answers or questions: Even minor hints give an unfair advantage. Discuss assessments only with Mr. Morales.

Bottom Line: All work must be original. Violations may result in reduced or no credit, an academic-integrity report, or further disciplinary action. If uncertain, ask Mr. Morales before submitting.

If unsure whether AI use is appropriate, ask Mr. Morales first; he is available to help students make responsible, ethical choices.

Permitted AI Uses

  • Clarifying difficult concepts
  • Breaking down and exploring new topics
  • Studying for quizzes or exams
  • Reviewing Python syntax
  • Debugging code (students must also show code versions before using AI to demonstrate original work)

Prohibited AI Uses

  • Generating full solutions for graded work
  • Writing code that is then submitted as original work
  • Answering reflection or documentation prompts for assignments

Improper AI use may result in a zero on the assignment, contacting parent/guardian, or more severe disciplinary action. Students are encouraged to use AI to learn more effectively, not to bypass the learning process.
